PlanQK Ontology Specification

Status of this Document

This document describes the PlanQK ontology. The document is still under development and content may be subject to change.

Abstract

This documentation describes the first version of the PlanQK ontology - an ontology for representing knowledge about quantum algorithms and their implementations. The PlanQK ontology serves as basis for the PlanQK platform - a collaborative platform for researchers and practitioners to support collection and development of knowledge on the field of quantum computing. The ontology is being developed as part of the PlanQK project.

Introduction back to ToC

The knowledge about quantum solutions for real world and research problems, the opportunities that these solutions offer for the industry, and the quantum resources needed to realize them is spread among different sources, such as publications, vendor-specific documentation and tutorials, or public code repositories. The PlanQK ontology accumulates the existing knowledge about the quantum computing algorithms and its implementation and provides it with semantics.
As a part of the collaborative platform the ontology has the following usage:
  • defines the machine-interpretable semantic description of the data and knowledge artifacts managed in the repositories,
  • serves as the foundation for the realization of various AI-based features from semantic faceted search and natural question answering to the semantic recommendation engine for supporting user’s decision by the selection of an appropriate quantum algorithm in a specific use case, and
  • encourage the knowledge evolving through expert exchange in view of the fact that the resulting findings of the expert discussions held on the platform can be documented in a semantically usable way leading to the collaborative development of new knowledge.

Data is a data item composed of data examples and it may be of a various level of granularity and complexity. External Vocabulary References back to ToC

The PlanQK ontology uses annotation properties to describe metadata of the ontology itself and its elements (classes, properties, individuals). Thereby a number of external vocabularies are referenced. This section provides the list of references.

Dublin Core terms

The PlanQK ontology uses following annotation properties to describe ontology itself:
  • ontology creator dcterms:creator,
  • ontology contributor dcterms:contributor , and
  • creation date of the ontologydcterms:created.
  • description of the ontology license conditions dcterms:license,
  • a name given to the onotlogy dcterms:title.
To reference a literature that motivated the inclusion of a specific ontology element, dcterms:source is used.

SKOS Simple Knowledge Organization System Reference

  • skos:example is used to illustrate the usage of the ontology element,
  • skos:prefLabel is used to store the preferred label of a specific ontology element (note that an element can have only one preferred label per language),
  • skos:altLabel is used to store the alternative labels of a specific ontology element, such as synonyms, abbrevations or acronyms.

VANN: A vocabulary for annotating vocabulary descriptions

The annotation property vsnn:preferredNamespacePrefix is used to specify the preferred namespace prefix when using elements of the PlanQK ontology in an XML document.

SemWeb Vocab Status ontology

Annotation properties sw:status and sw:term_status store the status of the onotology and ontology elements respectively.

FAIRsharing Subject Ontology (SRAO)

SRAO builds a hierarchy of academic disciplines and covers all of the humanities and natural sciences. The ontology is reused for describing domains in which a quantum application cab be applied or from which learning data is collected.

MEX Algorithm Ontology

The concept OntoQC_N00261 - Learning Method and its subclasses are derived from `http://mex.aksw.org/mex-algo#LearningMethod`.
